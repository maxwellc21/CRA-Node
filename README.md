# Customer Review App (Microservices + MVC)

A complete **Customer Review App** built with **Node.js, Express, MongoDB, Docker, and JWT authentication** using a **microservices architecture** with **MVC** inside each service.

This project is aligned to the IS312 capstone idea of a system where users can:
- register and log in
- browse products/services
- write reviews only when authenticated
- view their profile
- update and delete only their own reviews

## Architecture

Services:
- **api-gateway** - single entry point for browser and API traffic
- **auth-service** - register, login, token validation
- **product-service** - manage and list products/services
- **review-service** - create, update, delete, and list reviews
- **user-service** - user profile and user review summary
- **frontend** - static Bootstrap web UI

Each backend service uses MVC:
- `models/`
- `controllers/`
- `routes/`
- `middlewares/`
- `config/`

## Tech Stack
- Node.js
- Express.js
- MongoDB + Mongoose
- JWT
- Docker Compose
- Bootstrap 5
- Vanilla JavaScript frontend

## Main Features
- User registration and login
- Protected review posting
- Product browsing
- Review listing by product
- Profile page with user information and authored reviews
- Users can update/delete only their own reviews
- Seed data for demo products/services

## How to Run

### 1. Start everything
```bash
docker compose up --build
```

### 2. Open the app
```text
http://localhost:8080
```

## Default Ports
- Gateway: `8080`
- Auth Service: `4001`
- Product Service: `4002`
- Review Service: `4003`
- User Service: `4004`
- Frontend: served through gateway
- MongoDB: `27017`

## Seeded Demo Products
The product service seeds a few products automatically when it starts.

## API Summary

### Auth
- `POST /api/auth/register`
- `POST /api/auth/login`
- `GET /api/auth/verify`

### Products
- `GET /api/products`
- `GET /api/products/:id`
- `POST /api/products/seed`

### Reviews
- `GET /api/reviews/product/:productId`
- `POST /api/reviews`
- `PUT /api/reviews/:id`
- `DELETE /api/reviews/:id`
- `GET /api/reviews/user/:userId`

### Users
- `GET /api/users/me`
- `GET /api/users/:userId/reviews`

## Notes
- Every service uses its own MongoDB database for better microservice separation.
- The API gateway forwards requests to the appropriate internal service.
- JWT is generated by the auth service and validated by downstream services using the shared secret.
